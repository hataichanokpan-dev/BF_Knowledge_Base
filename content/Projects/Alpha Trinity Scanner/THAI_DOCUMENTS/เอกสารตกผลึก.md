### 2. การค้นพบสำคัญ

#### CAUTION Zone = Sweet Spot ⭐

```
ผลตอบแทนเฉลี่ย 6 เดือน:
├── CAUTION (20-50% gap)  → +16.59%  🏆 ดีที่สุด
├── ACCEPTABLE (<20%)     → +13.63%  ✅ ดี
└── AVOID (>50%)          →  -2.35%  ✅ ปกป้องทุน
```

**ความหมาย**: "Quality at Reasonable Price" ดีกว่า "Deep Value" ในตลาดไทย
- ถูกเกินไป = Value Trap (ถูกละเลย, ไม่มี growth)
- แพงเกินไป = Priced for Perfection (เสี่ยงผิดหวัง)
- พอดี = Institutional Favorites (คุณภาพ + growth ที่ราคาพอเหมาะ)

#### Data Triangulation = Competitive Moat

**ปัญหา**: yfinance คืนค่า EBIT = 0 สำหรับหุ้นไทยทุกตัว

**วิธีแก้**: ดึงข้อมูลโดยตรงจากรายงานการเงิน
- Income Statement: EBIT, Revenue, Net Income
- Balance Sheet: Debt, Cash, Equity
- Cash Flow: Operating Cash Flow, CapEx

**Impact**: นี่คือ "cost of entry" สำหรับความจริงในตลาด EM

## ช่องว่างวิจัย (Research Gaps)

### 1. Time-Series Regime Testing

**คำถาม**: Expectation gap ทำงานต่างกันในแต่ละ Regime ไหม?

| Regime | Liquidity | Growth | คาดว่า Gap ทำงานอย่างไร |
|--------|-----------|--------|----------------------|
| GOLDEN_LOCK | High | High | CAUTION leads อย่างมีนัย |
| GROWTH_SLOWDOWN | High | Low | Defensive sectors ดีกว่า |
| LIQUIDITY_CRUNCH | Low | High | Quality focus |
| STAGFLATION | Low | Low | Survival mode |

**วิธีวิจัย**:
1. แบ่งข้อมูลย้อนหลังตาม regime (ถ้ามีข้อมูล)
2. ทดสอบสหสัมพันธ์ในแต่ละ regime
3. ปรับ signal thresholds ตาม regime

### 2. Portfolio Construction Layer

**จาก Signal Validation → ไป Portfolio Strategy**

| คำถาม | วิธีวิจัย |
|--------|------------|
| กี่หุ้นในพอร์ตโฟลิโอ? | Test 10, 15, 20, 30 stocks |
| Rebalance เมื่อไร? | Test monthly, quarterly, ad-hoc |
| Position Sizing อย่างไร? | Equal weight vs Risk parity vs Kelly |
| Transaction Costs โหดแค่ไหน? | Add slippage + commission model |

### 3. Gap Decomposition

**แยก Gap ออกเป็นส่วนประกอบ**

```
Total Gap = f(Growth Gap, Margin Gap, ROIC Gap, WACC Gap)
```

**คำถาม**: ส่วนประกอบไหน predictive ที่สุด?

**วิธีวิจัย**:
1. Isolate each component
2. Test predictive power แยกกัน
3. Build ensemble model

### 4. Competitive Benchmark Study

**เปรียบเทียบกับ Signals อื่น**

| Signal | Expected Alpha |
|--------|----------------|
| Expectation Gap (เรา) | ? |
| P/E Ratio | ? |
| P/B Ratio | ? |
| Momentum | ? |
| Analyst Target Spread | ? |

**วิธีวิจัย**:
1. Run ทุก signals บน dataset เดียวกัน
2. Compare risk-adjusted returns (Sharpe, Sortino)
3. Regression: Expectation Gap ให้ incremental alpha หรือไม่?

### 5. Out-of-Sample & Cross-Market

**ตอบคำถาม: Specific to Thailand หรือ Generalizable to EMs?**

**วิธีวิจัย**:
1. Phase 1: Test on unseen Thai periods (roll-forward)
2. Phase 2: Test on ASEAN peers (Singapore, Malaysia, Indonesia)
3. Phase 3: Identify อะไร universal vs อะไร market-specific

### 6. Narrative-to-Number Diagnostics

**เชื่อมโดยการ Narrative กับ Numbers**

**คำถาม**: "Story inflation" อธิบาย extreme gaps ได้ไหม?

**วิธีวิจัย**:
1. Collect analyst/management narratives
2. Code ความ "bullish/bearish" ด้วย NLP
3. Test: Bullish stories → Higher gaps?

### 7. Failure-Mode Map

**เมื่อไหร่ High Gap ≠ Poor Returns?**

**Exceptions**:
- Real Options cases (high growth potential)
- Structural winners (competitive moat)
- Policy shocks (regulatory changes)

**วิธีวิจัย**:
1. Identify cases ที่ AVOID แต่ returns ดี
2. Find common patterns
3. Add filter ลง model

## บทเรียนสำคัญ

### 1. "Better Roughly Right Than Precisely Wrong"

**ปัญหา**: yfinance data = broken (EBIT = 0)

**การแก้**: Triangulate จากหลาย sources
- ไม่ต้อง perfect data แต่ต้อง "ใช้การได้"
- Trade-off: Coverage vs Accuracy

### 2. CAUTION Zone คือ Sweet Spot

**ผลลัพธ์**: 20-50% gap ให้ผลตอบแทนดีที่สุด

**คำอธิบาย**:
- Institutional appetite สำหรับ "quality at reasonable price"
- เกินไป = value trap, ไม่ถึง = growth trap
- พอดี = momentum + quality

### 3. Validation ต้อง Forward, Not Backward

**ปัญหา**: ไม่มี Point-in-Time data สำหรับ Thai stocks

**การแก้**: Forward tracking 12-24 เดือน
- Pre-register rules
- Track monthly
- Publish results (transparency)

### 4. Documentation = Reproducibility

**เป้าหมาย**: คนอื่นสามารถ reproduce ได้

**สิ่งที่ทำ**:
- 12+ documentation files
- Evidence matrix
- Assumption ledger
- API reference
